1.3签名规则

所有参数均可参与签名，不是必有的参数若值为空则不参与签名，若值不为空，则必须参与签名。参数签名之前，按照参数名ascll排序之后，签名格式如下：

 

签名串=MD5(amount=100&bankCode=4&clientIp=168.192.1.0&extend=扩展内容&goodsDetail=商品详情&goodsName=商品名称&mchId=商户号&notiryUrl=http://xxx.com/notifyUrl&orderNo=订单号&payTime=支付时间&returnUrl=http://xxx.com/returnUrl&signType=MD5+key)

 

例：key= 1241234123421343

签名串=MD5(amount=100&bankCode=4&clientIp=168.192.1.0&extend=扩展内容&goodsDetail=商品详情&goodsName=商品名称&mchId=商户号¬iryUrl=http://xxx.com/notifyUrl&orderNo=订单号&payTime=支付时间&returnUrl=http://xxx.com/returnUrl&signType=MD51241234123421343)

 

如果是Json网关则返回json如下：

{

"RState":"1",

"RMsg":"请求成功",

"Data":"http://url"

}

1.5验签规则

排除sign以外，所有不为空的参数都参与签名；

签名规则，同上1.3(签名规则)

之后对sign的值进行对比，签名相同则表示验签成功（商户可以做一些处理），否则验签失败。

验签成功请返回上游SUCCESS
